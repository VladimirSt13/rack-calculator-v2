## 🎯 Стратегия разработки

### 1️⃣ Сначала backend

**Почему:**

Backend — это «ядро» системы. Все функции, включая регистрацию, RBAC и аудит, будут использоваться и фронтендом, и будущими модулями.

Если сразу сделать фронтенд, а бекенд ещё «сырой», фронтенд будет зависеть от моков и придётся потом переписывать интеграцию.

Базовые API — регистрация, логин, роли — можно протестировать и утвердить до дизайна фронтенда.

**Что сделано на первом этапе бекенда:**

- ✅ Настроен проект (TS + Node + Express)
- ✅ Модуль users + auth (Register, Login, Reset Password)
- ✅ JWT Service + Auth Middleware
- ✅ Структура для RBAC и Audit
- ⚠️ Тесты — не созданы
- ⚠️ Email verification — без реализации отправки

**Результат:** рабочий backend, готовый принимать запросы с фронтенда.

---

### 2️⃣ После базового backend — frontend

**Почему:**

Теперь фронтенд может работать с настоящим API, а не с моками.

Фокус можно сделать на UI/UX, формах регистрации, логине, просмотре ролей и истории действий.

Фронтенд строится поверх стабильного ядра и легко интегрируется с будущими бизнес-модулями.

**Что нужно сделать на фронтенде:**

- ⏳ Настроить проект (React + TS + Vite) — структура готова
- ⏳ Настроить роутинг, формы и валидацию (Zod + React Hook Form)
- ⏳ Страницы: логин, регистрация, профиль
- ⏳ Настроить state management (Zustand / TanStack Query)
- ⏳ Подключить backend API для взаимодействия

**Результат:** UI, полностью связанный с backend, готовый для расширения под бизнес-функции.

---

### 3️⃣ После этого — параллельная разработка

Когда ядро готово и фронтенд умеет работать с API:

- Можно параллельно добавлять новые модули backend: Rack, Battery, Export
- Фронтенд развивает интерфейсы для этих функций
- Используется та же структура use-case → domain → repository → controller

## 🗺️ Дорожная карта спринтов

### Sprint 1 — Backend: фундамент ядра ✅

**Статус:** ✅ **ЗАВЕРШЁН**

**Выполнено:**

- Инициализация проекта (Node + TypeScript + Express)
- Настройка линтинга, тестов, конфигов (ENV, DB, JWT)
- Модуль users:
  - Entity: User (Prisma schema)
  - Repository: UserRepository
  - Use-cases: registerUser, loginUser, resetPassword, verifyEmail
- Auth модуль:
  - JwtService (access + refresh tokens)
  - Auth middleware
  - Controller: /api/auth/\* маршруты
- ⚠️ Тесты: не созданы
- ⚠️ RBAC и Audit: структура готова, без реализации

**Результат:** backend с регистрацией, логином, сбросом пароля, JWT работает.
