# 🗂️ План разработки приложения

## **Фаза 1 — Подготовка и фундамент**

**Цель:** создать основу для безопасного и расширяемого приложения.

**Что будет сделано:**

1. Настройка проекта и серверной платформы
2. Подключение базы данных
3. Настройка окружения и инструментов разработки
4. Подготовка структуры приложения для будущих модулей

**Что получит клиент:**

* Рабочее приложение с базовой архитектурой
* Возможность безопасного добавления новых функций

---

## **Фаза 2 — Пользователи и аутентификация**

**Цель:** дать пользователю возможность зарегистрироваться и безопасно заходить в систему.

**Что будет сделано:**

1. Регистрация пользователей
2. Вход в систему (логин) с токенами безопасности
3. Верификация email
4. Сброс пароля

**Что получит клиент:**

* Рабочий процесс регистрации и входа
* Безопасное хранение данных пользователя

---

## **Фаза 3 — Роли и права (RBAC)**

**Цель:** создать систему прав для разных пользователей.

**Что будет сделано:**

1. Создание ролей и разрешений
2. Ограничение доступа к функциям в зависимости от роли
3. Проверка прав на действия в приложении

**Что получит клиент:**

* Гибкая система управления пользователями
* Возможность легко добавлять новые функции и ограничивать доступ

---

## **Фаза 4 — Аудит действий**

**Цель:** фиксировать важные действия пользователей и системы.

**Что будет сделано:**

1. Логирование действий (создание, редактирование, удаление)
2. Возможность просмотра истории действий
3. Асинхронное сохранение логов для высокой производительности

**Что получит клиент:**

* Контроль всех ключевых действий в приложении
* Инструмент для анализа работы системы и безопасности

---

## **Фаза 5 — Базовые бизнес-функции (Rack & Battery)**

**Цель:** создать первую полезную функциональность для работы с задачами/моделями.

**Что будет сделано:**

1. Расчёт и подбор «стеллажей» (первый инструмент)
2. Подбор «батарей» и их совместимость
3. Сохранение конфигураций (комплектов)

**Что получит клиент:**

* Рабочая основная функциональность
* Возможность сохранять и использовать результаты расчётов

---

## **Фаза 6 — Вспомогательные функции**

**Цель:** сделать приложение удобным и полнофункциональным.

**Что будет сделано:**

1. Экспорт данных (Excel / PDF)
2. Версии и ревизии сохранённых комплектов
3. Восстановление удалённых элементов (soft delete / restore)
4. Фильтрация и пагинация данных

**Что получит клиент:**

* Удобство работы с результатами
* Контроль версий и данных
* Возможность делиться и экспортировать отчёты

---

## **Фаза 7 — Тестирование и деплой**

**Цель:** подготовить приложение для продакшен и безопасного использования.

**Что будет сделано:**

1. Полное тестирование всех функций
2. Настройка окружений: dev, staging, prod
3. Настройка CI/CD и деплоя через Docker
4. Документация API для интеграций

**Что получит клиент:**

* Рабочее, стабильное и безопасное приложение
* Возможность масштабирования и добавления новых инструментов

---

## ✅ Итоговая стратегия

* **Сначала фундамент:** пользователи, роли, аудит
* **Потом бизнес-функции:** расчёты, подбор компонентов, сохранение комплектов
* **В конце — удобство и экспорт:** версии, фильтры, отчёты
* Система строится модульно и безопасно, каждую новую функцию можно добавлять без переделки ядра
